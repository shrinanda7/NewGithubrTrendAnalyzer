# GitHub Trend Analyzer

This is the main project directory for the GitHub Repository Trend Analyzer, a Big Data pipeline that uses Apache Spark to process large-scale GitHub event data and visualize it via an interactive Flask dashboard.

## How to Run for Demonstration

To start the dashboard and view the visualizations, run the following command from this root directory:

```bash
python src/dashboard/app.py
```
Then, open your browser and navigate to `http://127.0.0.1:5000`.

---

## Directory Structure

*   `data/`: Stores the raw JSON dataset (`github_events.json`).
*   `src/`: Contains all the Python source code for data generation, Spark analytics, and the Flask web dashboard.
*   `output/`: Contains the processed JSON results generated by the Spark analyzer. These files are read by the dashboard.

## Explanation of 0 Records Cleaned
In the performance timing, you might notice `0 records cleaned`. This is because the dataset generated by our Python script (`src/data_generator.py`) is already perfectly formatted with no missing critical fields (`repo.language`, `created_at`, `repo.name`). Thus, Spark's filtering step didn't need to discard any malformed rows. If you use raw, real-world GitHub Archive data, this number would likely be much higher.
