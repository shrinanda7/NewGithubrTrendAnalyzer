# Output Data Directory (`output/`)

This directory contains the final, processed artifacts generated by the PySpark analytical pipeline (`src/spark_analyzer.py`).

The raw JSON events (over 2 million rows, ~1GB) are reduced into these highly aggregated, lightweight JSON files. This architecture allows the Flask dashboard to render complex analytics instantly without querying a database on the fly.

## Files Generated

1.  **`1_language_popularity.json`**: Contains aggregated counts of events grouped by programming language and year.
2.  **`2_top_repos_by_stars.json`**: A deduplicated list of top repositories sorted by their maximum observed star count.
3.  **`3_topic_trends.json`**: Arrays of GitHub topics "exploded" from repo metadata and counted per year to track developer trends (e.g., the rise of "ai", "machine-learning").
4.  **`4_yoy_growth.json`**: Contains year-over-year percentage growth for languages, calculated using Spark SQL Window functions.
5.  **`5_star_distribution.json`**: Statistical distribution (average, median) of stars across different programming languages.
6.  **`6_activity_index.json`**: A custom composite metric showing the most consistently active repositories.
7.  **`7_ecosystem_health.json`**: A complex, normalized score (0-100) combining repository count, stars, forks, and topic density.
8.  **`8_user_profiles.json`**: Analyzed user behavior from historical data. (Note: The dashboard primarily uses the live GitHub API for this now, but the Spark phase still processes it as a demonstration of Big Data profiling).
9.  **`performance_timing.json`**: Contains precise benchmarking metrics tracking the exact time taken for the "Mapper" (transformations) and "Reducer" (actions) phases of each Spark job.
